# KG tools

Knowledge graph and data-oriented tools.

## Tools

- [Shapes Constraint Language (SHACL)](./shapes-constraint-language-shacl.md) — A W3C language for validating RDF graphs against a set of conditions. These conditions are provided as shapes and other constructs expressed in the form of an RDF graph.
- [Apache Jena SHACL](./apache-jena-shacl.md) — Apache implementation of the W3C SHACL. It implements SHACL Core and SHACL SPARQL Constraints.
- [pySHACL](./pyshacl.md) — Python-based SHACL validator for RDF graphs.
- [SHACL Playground](./shacl-playground.md) — A constraint validator for the Shapes Constraint Language, written in JavaScript.
- [SHACL Play!](./shacl-play.md) — Free suite of tools from Sparna to work with SHACL.
- [Shape Expressions](./shape-expressions.md) — A structural schema language for validation, traversal and transformation of RDF graphs.
- [ShEx.js](./shexjs.md) — Shape Expressions validator for JavaScript.
- [RDFUnit](./rdfunit.md) — Rule-based validation framework using SPARQL test patterns to detect structural issues and missing links.
- [F-UJI](./f-uji.md) — A web service to programatically assess FAIRness of research data objects based on metrics developed by the FAIRsFAIR project.
- [FAIR-Checker](./fair-checker.md) — A web and command line tool to assess FAIRness of web resources.
- [FAIR Evaluator](./fair-evaluator.md) — Resources and guidelines to assess the FAIRness of digital resources.
- [FAIR-Shake](./fair-shake.md) — Combines automated FAIR checks with human-evaluated rubrics.
- [TripleCheckMate](./triplecheckmate.md) — Crowdsourcing platform for human validation of RDF triples selected automatically.
- [CrowdHeritage](./crowdheritage.md) — Open crowdsourcing platform for cultural heritage institutions to run campaigns in which contributors enrich and validate digitised cultural heritage metadata by adding annotations or reviewing existing ones.
- [Europeana methodology for validating enrichments](./europeana-methodology-for-validating-enrichments.md) — Methodology used by the Europeana Foundation to validate metadata enrichment efforts before acceptance. It combines documentation of the enrichment context, representative sampling, manual or semi-automated assessment of sample outputs, validation against agreed quality requirements, and, where relevant, calibration of confidence thresholds and iterative improvement of the enrichment process.
