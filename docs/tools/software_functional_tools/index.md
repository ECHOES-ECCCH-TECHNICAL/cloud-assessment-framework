# Software functional tools

Functional testing or validation tools.

## Tools

- [ISO/IEC 25040 – Evaluation Process (SQuaRE)](./isoiec-25040-evaluation-process-square.md) — Defines a standard process model for planning, executing and concluding software quality evaluations, ensuring that assessments are systematic, repeatable and auditable.
- [Requirements Traceability Matrix (RTM)](./requirements-traceability-matrix-rtm.md) — Provides a structured artefact to map functional requirements to implementation and test artefacts, supporting monitoring of functional completeness and correctness.
- [Automated functional test suites + coverage tools (e.g. JUnit/pytest/Jest with JaCoCo, coverage.py, Istanbul)](./automated-functional-test-suites-coverage-tools-eg-junitpytestjest-with-jacoco-coveragepy-istanbul.md) — Provides automated execution and measurement of functional tests to verify correctness of implemented behaviour and quantify test coverage of functional requirements.
- [OpenAPI-based API contract testing (e.g. Dredd, Schemathesis)](./openapi-based-api-contract-testing-eg-dredd-schemathesis.md) — Provides automated validation of API behaviour against formally specified OpenAPI contracts to ensure functional correctness and robustness at the interface level.
- [Property-based / fuzz testing libraries (e.g. Hypothesis; Schemathesis property-based mode)](./property-based-fuzz-testing-libraries-eg-hypothesis-schemathesis-property-based-mode.md) — Provides automated generation and execution of test cases based on properties and invariants to detect functional defects under a wide range of input conditions.
- [Task-based user test protocol](./task-based-user-test-protocol.md) — Provides a scenario-based testing protocol to evaluate whether users can successfully complete representative tasks using available system functions.
