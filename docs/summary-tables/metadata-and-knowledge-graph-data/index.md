# Metadata and Knowledge Graph Data

| | |
| --- | --- |
| **Primary section** | **Section 6.2 — Evaluation of Metadata and Knowledge Graph Data** |
| **Evaluation basis** | <ul><li>Metadata records</li><li>Knowledge graph data</li><li>Schemas, application profiles, validation rules or equivalent documentation</li><li>Identifiers, labels, properties, links, licences and rights information</li><li>Evidence produced through validation, profiling or review procedures</li></ul> |
| **Core dimensions / criteria** | <ul><li>Structural Completeness</li><li>Property Coverage</li><li>Multilingual Labeling Coverage</li><li>Foundational and Format Consistency</li><li>Accessibility and Connectivity</li><li>Licensing (Rights)</li><li>Conformance to Standards</li></ul> |
| **Main instruments** | <ul><li>Metadata and knowledge graph validation tools</li><li>Schema, profile or constraint-based checks</li><li>Completeness and coverage analysis</li><li>FAIRness and interoperability-oriented checks</li><li>Licence, rights and provenance review</li><li>Expert or curator review where semantic interpretation is required</li></ul> |
| **Expected output** | <ul><li>Structured evidence on metadata or knowledge graph quality</li><li>Assessment of completeness, consistency, accessibility, connectivity, licensing and standards conformance</li><li>Identification of gaps, inconsistencies or improvement needs</li><li>Evidence that can support quality review and later indicator computation</li></ul> |
| **What resource providers should prepare** | <ul><li>Access to the metadata or knowledge graph resource</li><li>Documentation of the data model, schema, ontology, application profile or mappings used</li><li>Information on identifiers, labels, links, licences, rights and provenance</li><li>Available validation reports or quality-control evidence</li><li>Known limitations or pending improvements</li></ul> |
