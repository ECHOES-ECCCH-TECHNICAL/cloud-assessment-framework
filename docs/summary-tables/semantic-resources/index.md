# Semantic Resources

| | |
| --- | --- |
| **Primary section** | **Section 6.5 — Evaluation of Semantic Resources** |
| **Evaluation basis** | <ul><li>Ontology, vocabulary, taxonomy, thesaurus or semantic mapping</li><li>Documentation and usage guidance</li><li>Scope, purpose and intended use</li><li>Mappings or alignments with other semantic resources</li><li>Evidence on understandability, interoperability and reuse</li></ul> |
| **Core dimensions / criteria** | <ul><li>Usability and Understandability</li><li>Interoperability and Compatibility</li></ul> |
| **Main instruments** | <ul><li>Documentation review</li><li>Ontology or vocabulary quality checks</li><li>Consistency and compatibility checks</li><li>Mapping and alignment review</li><li>Expert review</li><li>User or community feedback where appropriate</li></ul> |
| **Expected output** | <ul><li>Evidence on usability, understandability, interoperability and compatibility</li><li>Identification of documentation gaps, ambiguous concepts or alignment issues</li><li>Assessment evidence supporting review and improvement of the semantic resource</li><li>Recommendations for clarification, extension or better integration</li></ul> |
| **What resource providers should prepare** | <ul><li>Machine-readable semantic resource file or access point</li><li>Human-readable documentation</li><li>Statement of scope and intended use</li><li>Examples of use where available</li><li>Mappings or alignments with relevant standards, vocabularies or ontologies</li><li>Versioning or maintenance information</li></ul> |
