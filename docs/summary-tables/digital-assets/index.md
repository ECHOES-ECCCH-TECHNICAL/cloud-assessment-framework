# Digital Assets

| | |
| --- | --- |
| **Primary section** | **Section 6.3 — Evaluation of Digital Assets** |
| **Evaluation basis** | <ul><li>Digital asset files or access links</li><li>Technical metadata</li><li>Digitization, acquisition or processing documentation</li><li>Paradata where available</li><li>Use-case information relevant to assessing fitness for purpose</li></ul> |
| **Core dimensions / criteria** | <ul><li>No single uniform core dimension set is defined for all digital asset types in D9.1</li><li>Asset-type-specific quality criteria should be refined through later CAF implementation and the wiki</li><li>Data usage, where relevant to access, uptake and reuse</li></ul> |
| **Main instruments** | <ul><li>Asset-type-specific quality assessment methods</li><li>Technical metadata and paradata review</li><li>Expert review where automated assessment is insufficient</li><li>Reference-based or no-reference quality assessment approaches where appropriate</li><li>Use-case-specific assessment criteria</li></ul> |
| **Expected output** | <ul><li>Evidence on the quality and fitness for purpose of the digital asset</li><li>Identification of asset-specific quality issues affecting reuse or interpretation</li><li>Documentation of relevant technical, acquisition or processing context</li><li>Recommendations for further assessment or improvement where needed</li></ul> |
| **What resource providers should prepare** | <ul><li>Access to the digital asset or representative sample</li><li>Technical metadata</li><li>Digitization, acquisition or processing documentation</li><li>Information on intended use cases</li><li>Known quality limitations or uncertainty information</li><li>Existing quality-control reports where available</li></ul> |
