# Software

| | |
| --- | --- |
| **Primary section** | **Section 6.6 — Evaluation of Software** |
| **Evaluation basis** | <ul><li>Software codebase or repository</li><li>Software documentation</li><li>Licence and reuse information</li><li>Installation, deployment or execution information</li><li>Testing, release, maintenance or security evidence where available</li><li>User-facing evidence where usability is assessed</li></ul> |
| **Core dimensions / criteria** | <ul><li>Compatibility</li><li>Open-Source Software</li><li>FAIRness</li><li>Security</li><li>Usability</li></ul> |
| **Main instruments** | <ul><li>Repository and documentation review</li><li>Software quality checklists</li><li>Static analysis or code-quality tools where appropriate</li><li>Testing and release evidence review</li><li>Security-oriented checks</li><li>FAIR software assessment instruments</li><li>User feedback or pilot-based usability evaluation</li></ul> |
| **Expected output** | <ul><li>Software quality self-assessment evidence</li><li>Evidence on compatibility, openness, FAIRness, security and usability</li><li>Identification of documentation, maintainability, security or reuse issues</li><li>Evidence supporting ECHOES quality review and improvement planning</li></ul> |
| **What resource providers should prepare** | <ul><li>Repository URL or source-code package</li><li>Licence information</li><li>README, installation and usage documentation</li><li>Information on dependencies, releases and maintenance status</li><li>Available test, quality or security reports</li><li>Description of intended users and use context</li></ul> |
